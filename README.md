# Vienna Bike Availability Prediction

This repository contains the final project for predicting the number of available bicycles at bike-sharing stations in Vienna.

## Project Files

- `project_notebook.ipynb`: Complete project workflow, including data exploration, feature engineering, validation, model comparison, hyperparameter tuning, final model training, interpretation, and submission generation.
- `submission.csv`: Final predictions generated by the notebook.

## Data

The project uses the provided `train.csv` and `test.csv` datasets.

Due to file size limitations, the original course datasets are not included in this repository. To reproduce the results, place both files in the same folder as the notebook:

```text
train.csv
test.csv
project_notebook.ipynb
```

No external data were used in this project.

## Methodology

The workflow includes:

- Exploratory data analysis
- Chronological validation
- Calendar and time feature engineering
- Station-level and global time-profile features
- Feature selection
- Comparison of XGBoost, CatBoost, and LightGBM
- XGBoost hyperparameter tuning
- Bucket-based post-processing

## Final Model

The final model is an XGBoost regression model trained using:

- 23 selected features
- A `log1p` transformation of the target variable
- 448 boosting rounds
- Bucket-based calibration of the final predictions

## How to Run

1. Place `train.csv` and `test.csv` in the same directory as `project_notebook.ipynb`.
2. Open the notebook in Jupyter Notebook or JupyterLab.
3. Run all cells from top to bottom.
4. The notebook will generate a file named `submission.csv`.
