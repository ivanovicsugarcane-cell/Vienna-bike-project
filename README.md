## Files

- `project_notebook.ipynb`: Main project document containing data exploration, feature engineering, validation, model development, interpretation, and final submission generation.
- `project.R`: Standalone R script that reproduces the final model and generates `submission.csv`.
- `submission.csv`: Final prediction file.
- `scripts/`: Experiment code for feature selection, learner comparison, and hyperparameter tuning.
- `results/`: Validation scores, model-comparison tables, selected parameters, and other outputs generated by the experiment scripts.

The experiment scripts are included for reproducibility but are not required to generate the final submission.


```text
vienna-bike-project/
├── train.csv
├── test.csv
├── project_notebook.ipynb
├── project.R
└── ...
```

The filenames must remain exactly `train.csv` and `test.csv`.

No external data were used.

## Validation

The data were split chronologically because the test period follows the training period.

Three expanding-window folds with 14-day validation periods were used for feature selection, learner comparison, and hyperparameter tuning. Target-based profile features were calculated using only the training portion of each fold to prevent data leakage.

An additional 48-day outer period was used only for final temporal confirmation and not for hyperparameter selection.

Performance was evaluated using RMSLE.

## How to Run the Notebook

1. Place `train.csv` and `test.csv` in the repository root.
2. Open `project_notebook.ipynb` in Jupyter Notebook or JupyterLab.
3. Restart the kernel and run all cells from top to bottom.
4. The notebook will generate `submission.csv` in the repository root.

## How to Run the Standalone R Script

Run the following command from the repository root:

```r
source("project.R")
```

The script will generate `submission.csv` in the repository root.

Both `project_notebook.ipynb` and `project.R` are intended to reproduce the same final submission. Running either workflow may overwrite the existing local `submission.csv`.

## Experiment Scripts

The development experiments can be run from the repository root:

```r
source("scripts/01_feature_selection.R")
source("scripts/02_learner_comparison.R")
source("scripts/03_hyperparameter_tuning.R")
```

Their outputs are saved automatically under `results/`. These experiments may require substantial computation time and do not need to be rerun to generate the final submission.
